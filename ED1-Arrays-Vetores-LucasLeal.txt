TAREFA 4: Para entender por que um ⁠int⁠ se comporta diferente de um ⁠array⁠ quando passado para uma função, imagine a Pilha de Chamadas (Call Stack) como uma série de "pastas" ou "salas" onde cada função trabalha.
          Variáveis Simples (⁠int x⁠)
          Quando você passa uma variável simples (por valor), o computador cria uma cópia do valor daquela variável e a coloca dentro da "pasta" da função que a chamou.
          Se a função ⁠normalizar⁠ alterar ⁠x⁠, ela estará alterando apenas a cópia que está na sala dela. A variável original, na sala da função ⁠main⁠, permanece intacta. É como se a função tivesse sua própria folha de papel com o mesmo número escrito nela.
          Arrays (⁠int v[]⁠)
          Arrays são grandes demais para serem copiados toda vez que chamamos uma função (isso tornaria os programas muito lentos). Por isso, o C não passa o array; ele passa apenas um endereço (como se fosse um "endereço de correspondência" ou um mapa indicando onde o array está guardado).
          Quando a função ⁠normalizar⁠ recebe esse endereço, ela não cria uma cópia dos dados. Ela olha para o endereço, vai até a "sala" onde o ⁠main⁠ guardou o array e altera os valores diretamente lá.
          Por isso, quando a função ⁠main⁠ volta a olhar para o seu array original, os dados já foram modificados.



--- EXERCICIO 4 ---

Tema: A Remoção de Tripulantes da ISS 

Tópico: Função remove_elemento, acesso fora dos limites no shift 

Contexto IA:  A ISS precisa remover um tripulante da lista de bordo quando alguém retorna à Terra. 
A IA implementou a função de remoção, mas ao remover o último elemento o programa apresenta 
comportamento inesperado. 

TAREFA 1: O código tenta acessar v[n].Em um vetor de tamanho n, os índices válidos vão de 0 a n - 1.Portanto, v[n] é um índice inválido
          (acesso fora dos limites ou out of bounds).O programa lerá o valor que estiver na memória logo após o final do vetor, o que 
          gera comportamento indefinido.


TAREFA 2: Com j < n ,o laço tenta puxar o elemento da frente para a posição atual até o último índice, caindo no erro .
          Com j < n - 1, o laço para no npenúltimo elemento.quando j = n - 2, o código executa v[n-2] = v[n-1].Como o último
          índice válido é n - 1,essa operação é segura e copia corretamente o último elemento para posição anterior.


TAREFA 3: // Código corrigido

int remove_tripulante(int k, int v[], int n) { 
    int j; 
    for (j = k; j < n - 1; j++) {  
        v[j] = v[j + 1];        
    } 
    return n - 1; 
}

          (k = 5, n = 8): o loop percorre j de 5 até 6. j = 5; v[5] = v[6]. j = 6: v[6] = v[7]. O elemento no índice 5 foi sobrescrito
          pelo índice 6, e o de 6 pelo de 7. O valor original de v[7] permanece lá, mas como a função retorna n - 1(7), o sistema passará
          a considerar apenas os índices de 0 a 6.


TAREFA 4: Significado: O "custo" (tempo de execução) é diretamente proporcional ao número de elementos que precisam ser deslocados.
          Comparação: Se o vetor tem 8 elementos, você faz poucos deslocamentos. Se tiver 8.000.000, o processador precisará realizar milhões
          de cópias na memória. O tempo cresce linearmente com o tamanho do vetor.Lista Encadeada: Em uma lista encadeada, para remover
          um elemento, você apenas altera o ponteiro do elemento anterior para apontar para o próximo. Não há deslocamento de elementos,
          tornando a remoção (se você já estiver no nó) uma operação O(1), ou seja, de custo constante, independentemente do tamanho da
          lista.

Tema:  O Gerador de Sinais do SETI 

Tópico: rand() / srand(), semente fixa gerando sequência idêntica 

Contexto IA:  O SETI usa um sistema simulado para gerar sequências de sinais pseudo-aleatórios 
para treinar detectores. A IA gerou o código, mas todos os elementos do vetor sempre têm o mesmo 
valor, o detector recebe dados inúteis.

TAREFA 1: Sim, todos os elementos do vetor terão o mesmo valor.A função srand(1) define o ponto de partida(semente) para geração de
          números. Como o srand(1) está dentro do loop, a semente está sendo redefinida para 1 a cada iteração.Consequentemente a chamada
          subsequente de rand() vai sempre retornar o primeiro número da sequência correspondente à semente 1, repetindo o mesmo resultado
          ifinitamente.


TAREFA 2: A "Semente" é um valor inicial passado para o algoritmo matemático que gera os números pseudo-aleatórios.É ela que dita
          qual será a sequência gerada.A recomendação de usar srand((unsigned)time(NULL)) uma única vez antes do loop serve para
          inicializar o gerador apenas no começo da execução.Se colocarmos dentro do loop,o gerador será reiniciado repetidamente.
          Além disso,como time(NULL) muda a cada segundo,chamar isso dentro de um loop muito rápido faria com que a semente fosse
          a mesma para várias iterações,gerando números repetidos.


TAREFA 3: Usar srand(time(NULL)) gera sequências diferentes a cada execução do programa porque a função time(NULL) retorna o tempo atual
          do sistema operacional em segundos.Como o tempo está sempre avançando, cada vez que você roda o programa, uma semente diferente
          é usada,resultando em uma sequência nova e imprevisível.

#include <stdio.h> 
#include <stdlib.h> 
#include <time.h> 
#define TAM 10 
 
int main() { 
    int sinais[TAM]; 
    int i; 
     srand(time(NULL)); 
    for (i = 0; i < TAM; i++) { 
                         
        sinais[i] = rand() % 256; 
    } 
 
    printf("Sinais capturados:\n"); 
    for (i = 0; i < TAM; i++) 
        printf("Sinal[%d] = %d\n", i, sinais[i]); 
 
    return 0; 
} 


TAREFA 4: Não, a função ⁠rand()⁠ não é suficiente para aplicações críticas como criptografia porque ela é previsível se a semente for descoberta.
          Diferença: PRNG (Pseudo-Random Number Generator) usa fórmulas matemáticas determinísticas para criar sequências que apenas parecem
